BLE UUID Proximity Test (Web Bluetooth)
=======================================

Что делает
----------
- Сайт генерирует BLE_UUID
- Пользователь вставляет BLE_UUID в "мобильное" приложение (симулятор)
- Симулятор рекламирует (advertising) BLE сервис с этим UUID
- Сайт сканирует BLE-эфир до 30 секунд
- Нашёл → ОК, иначе → ОШИБКА

Состав
------
- tests/web/ — статический сайт (Web Bluetooth API). Генерация UUID, сканирование.
- tests/mobile-sim-py/ — Python-симулятор "Android-приложения" для Windows, рекламирует BLE UUID как сервис.

Запуск сайта
------------
Нужен любой локальный web-сервер на localhost (Web Bluetooth требует secure context; localhost подходит).

Вариант 1 (Python, быстрый):

    cd tests/web
    python -m http.server 3002

Откройте в Chrome/Edge:

    http://localhost:3002

Вариант 2 (любой другой сервер, например `npx serve`), на ваше усмотрение.

Запуск симулятора (Windows, Python)
-----------------------------------
1) Установите зависимости:

    cd tests/mobile-sim-py
    pip install -r requirements.txt

2) Запустите с BLE_UUID (скопируйте с сайта):

    python ./ble_advertiser.py <BLE_UUID>

Пример:

    python ./ble_advertiser.py 12345678-1234-1234-1234-1234567890ab

Как тестировать
---------------
1) Откройте страницу tests/web, нажмите "Сгенерировать", скопируйте UUID.
2) Запустите симулятор с этим UUID (он начнёт рекламировать сервис).
3) На странице нажмите "Сканировать (30 сек)":
   - Если поддерживается `navigator.bluetooth.requestLEScan`, будет пассивный поиск и авто-детект.
   - Иначе откроется диалог `requestDevice` с фильтром по сервису UUID — выберите устройство симулятора.
4) При успехе увидите "ОК"; если не найдено за 30 сек — "ОШИБКА".

Примечания
----------
- На некоторых версиях браузеров `requestLEScan` может требовать включения флага:
  chrome://flags/#enable-experimental-web-platform-features
- Без `requestLEScan` сайт не видит список устройств пассивно; используется `requestDevice` (диалог выбора).
- Убедитесь, что Bluetooth включён, а симулятор запущен и близко к ПК.

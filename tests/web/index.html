BLE UUID Proximity Test (Web Bluetooth)

Поток:

  1. Сайт генерирует BLE_UUID.
  2. Пользователь вставляет BLE_UUID в Android-приложение (здесь симулятор для Windows).
  3. Приложение начинает рекламировать (advertising) этот BLE_UUID как сервис.
  4. Сайт сканирует Bluetooth в поисках этого BLE_UUID до 30 секунд.
  5. Если найдено — ОК, иначе — ОШИБКА.

Технически
  • Если доступно navigator.bluetooth.requestLEScan, используется пассивное сканирование и авто-детект сервиса.
  • Иначе — requestDevice({ filters: [{ services: [UUID] }] }) (понадобится выбрать устройство в системном диалоге).

Симулятор Android-приложения (Python, Windows)

Запустите Python-симулятор, который будет рекламировать BLE_UUID как BLE-сервис:

1) Убедитесь, что активировано ваше Python venv (если используете) и установите зависимости:
   cd C:\Users\healk\Desktop\QRCODE\tests\mobile-sim-py
   pip install -r requirements.txt

2) Запуск с вашим BLE_UUID:
   python .\ble_advertiser.py <BLE_UUID>

   Пример:
   python .\ble_advertiser.py 12345678-1234-1234-1234-1234567890ab
Убедитесь, что Bluetooth включён. Во время сканирования держите "телефон" (симулятор) рядом.
